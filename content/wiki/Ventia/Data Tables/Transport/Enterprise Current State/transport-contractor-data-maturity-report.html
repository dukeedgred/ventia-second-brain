Scoring Rubric

Each dimension is scored from 1 to 5. A score of 5 requires broad, reusable, active, and well-evidenced data. No score claims production certification or row-level data quality.

Assumptions, Risks, And Gaps

These constraints are deliberately visible because unsupported maturity claims would be misleading.

Assessment Boundary

  • Scores use documented schemas, table names, column counts, domains, and selected stakeholder notes.
  • They do not use row counts, actual refresh telemetry, Power BI usage logs, or direct data-quality profiling.
  • Where a table is identified as created by file upload UI, that is treated as weaker governance evidence unless other source details exist.

Known Exclusions

  • Maximo source-table documentation is deferred from this pass at the user's request.
  • Shared enterprise or staging schemas are not contractor scores.
  • Asset Vision source catalogues are evidence for source context, but not scored as contractors.

Decision Use

  • Use high-scoring contracts as candidates for reusable data patterns.
  • Use low-scoring contracts to prioritise source discovery, ownership, and active-feed validation.
  • Do not treat any score as a certification that a schema is production-ready for enterprise asset master use.
